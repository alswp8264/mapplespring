# Mapple Spring

메이플스토리 캐릭터 정보 조회 · 커뮤니티 웹 서비스

넥슨 「메이플스토리」 오픈 API를 기반으로, 게임 캐릭터의 스펙 정보를 검색·분석하고 랭킹·이벤트를 확인하며, 코디(룩북)·게시판으로 유저들이 소통하는 캐릭터 정보 + 커뮤니티 웹 서비스입니다.

## 기획 배경 및 목적

- 메이플스토리 유저는 자신/타인 캐릭터의 스탯·장비·유니온 등 '스펙'을 자주 확인하고 비교한다.
- 흩어져 있는 캐릭터 조회·랭킹·이벤트 정보를 한 사이트에서 보기 쉽게 제공하는 것이 목표.
- 여기에 코디 자랑(룩북), 공략 게시판 같은 커뮤니티 요소와 편리한 로그인을 더해 이용성을 높였다.

## 주요 특징

- **실제 넥슨 오픈 API 연동** — 캐릭터 기본정보·스탯·장비·코디·유니온·아티팩트·랭킹·공지 조회
- **3종 로그인** — 일반(아이디/비번), 소셜(구글·카카오 OAuth2), 얼굴 인식(정면·좌·우 3단계 등록)
- **게시판** — 카테고리(공략/전사/도적/마법사/해적/궁수), 이미지·동영상(최대 3분)·유튜브 첨부
- **운영 기능** — 게시글 신고 → 관리자 승인 시 글 삭제 + 작성자 1시간 글쓰기 제한(밴)

## 사용 대상

| 구분 | 설명 |
|---|---|
| 비회원 | 캐릭터 검색·랭킹·룩북·게시판 등 열람 |
| 회원(USER) | 즐겨찾기·스펙 히스토리·룩북 등록/좋아요·글/댓글 작성·얼굴 등록 |
| 관리자(ADMIN) | 회원·게시글·룩북·신고·공지 관리 |

## 기술 스택

- **Backend**: Java 21, Spring MVC(XML 설정), MyBatis + Spring Data JPA/Hibernate
- **DB**: Oracle XE
- **View**: JSP/JSTL
- **WAS**: Apache Tomcat 11 (포트 8088)
- **외부 연동**: 넥슨 오픈 API, Google/Kakao OAuth2, Python Flask 얼굴인식 서버(face_recognition, 포트 5001)

## 데이터베이스 구조

전체 13개 테이블로 구성되어 있습니다.

| 테이블 | 설명 |
|---|---|
| MEMBER_TB | 회원 정보(권한 role·소셜 연동·얼굴 인코딩·밴 만료시각 ban_until 포함) |
| BOARD_TB | 게시글(카테고리 category 포함) |
| COMMENT_TB | 게시글 댓글 |
| BOARD_FILE_TB | 게시글 첨부(이미지/동영상/유튜브) — file_type 로 구분 |
| REPORT_TB | 게시글 신고(상태 PENDING/DONE) |
| FAVORITE_TB | 회원의 캐릭터 즐겨찾기 |
| SPEC_HISTORY_TB | 캐릭터 스펙 스냅샷(공격력·보스뎀·방무·최종뎀) |
| LOOKBOOK_TB | 코디 공유(룩북) + 좋아요 수 |
| LOOKBOOK_LIKE_TB | 룩북 좋아요 내역 |
| ANNOUNCEMENT_TB | 관리자 공지사항 |
| NOTICE_THUMB_TB | 넥슨 이벤트/캐시샵 공지 썸네일 캐시 |
| MAPLE_ACCOUNT | 유니온 계정 캐시(JPA) |
| MAPLE_CHARACTER | 계정 소속 캐릭터 캐시(JPA) |

## 구현한 기능

- **캐릭터 조회**: 검색, 스탯/전투력(억·만 표기), 장비(잠재 등급), 코디(프리셋), 유니온, 아티팩트
- **랭킹**: 종합·유니온·무릉도장·더시드·업적 + 월드/직업 필터, 인기 캐릭터 이미지
- **공지**: 넥슨 이벤트·캐시샵(썸네일 캐시), 관리자 공지사항 게시
- **커뮤니티**: 룩북(등록·좋아요·코디분석), 게시판(카테고리·이미지·동영상·유튜브), 댓글
- **게시판 운영**: 게시글 신고 → 관리자 승인 시 삭제 + 작성자 1시간 밴(글·댓글)
- **회원**: 가입/로그인/마이페이지/정보·비번수정/탈퇴, 아이디·비밀번호 찾기
- **로그인 3종**: 일반, 소셜(구글·카카오), 얼굴 인식(정면·좌·우 3단계 등록 + 다각도 대조)
- **관리자**: 회원 검색·삭제, 게시글·룩북 관리, 신고 관리, 공지 작성 (관리자 전용 접근 보호)

## 실행 방법

### 1. Backend (Spring MVC on Tomcat)

1. Oracle XE에 DB 계정을 생성하고 `db_dump.sql`을 실행해 스키마/데이터를 구성합니다.
2. `src/main/resources/application.properties`에서 DB 접속 정보(계정/비밀번호 등)를 환경에 맞게 수정합니다.
3. Maven으로 빌드 후 Tomcat 11에 배포합니다. (기본 포트: `8088`)

### 2. 얼굴 인식 서버 (Flask)

```bash
cd face_server
python -u app.py
```

또는 루트의 `flask_start.bat` 실행 (기본 포트: `5001`)

### 3. 접속

```
http://localhost:8088/Mapple_Spring/
```
